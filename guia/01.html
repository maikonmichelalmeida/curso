Mux 4:1 do zero

Este guia começa do ponto mínimo: um RTL simples, um testbench simples, um filelist e um Makefile pequeno para lembrar como rodar VCS e Verdi no servidor.

Ainda não estamos fazendo Tcl, SDC, Design Compiler, DC_NXT ou Formality. Esses assuntos entram depois, quando este fluxo básico estiver confortável.

1. Estrutura criada

curso/01/
  README.md
  rtl/
    mux4.sv
  tb/
    tb_mux4.sv
  simulation/
    filelist.f
    Makefile

A separação é simples: rtl guarda o circuito, tb guarda o teste, e simulation guarda os comandos de simulação.

2. O circuito

O arquivo rtl/mux4.sv implementa um mux 4:1. O sinal sel escolhe qual entrada aparece na saída y.

sel saída y
2'b00d0
2'b01d1
2'b10d2
2'b11d3

Repare no uso de always_comb. Ele diz claramente que o bloco é combinacional. Mais tarde vamos comparar isso com estilos que podem gerar latch ou mismatch, mas agora só queremos o caminho correto.

3. O testbench

O arquivo tb/tb_mux4.sv instancia o mux, aplica quatro valores em sel e verifica se y corresponde à entrada esperada. 

verifica_mux(2'b00, d0);
verifica_mux(2'b01, d1);
verifica_mux(2'b10, d2);
verifica_mux(2'b11, d3);

Se tudo passar, o log deve mostrar TEST_RESULT: PASS.

4. O filelist

O arquivo simulation/filelist.f é a lista que o VCS lê. Por pedido nosso, o testbench aparece primeiro: 

../tb/tb_mux4.sv
../rtl/mux4.sv

Isso ajuda a visualizar que o top da simulação é o testbench. Ele instancia o DUT e conduz o teste.

5. O Makefile mínimo

O arquivo simulation/Makefile guarda comandos que você rodaria manualmente. Por enquanto, Makefile é só isso: nomes curtos para comandos.

Comando O que faz
make compchama o VCS e compila os arquivos do filelist
make simexecuta o simulador simv
make testlimpa, compila e simula
make wavesabre mux4.fsdb no Verdi
make cleanremove arquivos gerados

6. Ordem para rodar no servidor

cd ~/curso
git pull --ff-only origin main
cd 01/simulation
make help
make comp
make sim
make waves

Se as ferramentas ainda não estiverem carregadas, carregue antes de compilar: 

module load vcs/W-2024.09-SP2-3 verdi/W-2024.09-SP2-6

7. O que observar

  1. Depois de make comp, leia comp.log.
  2. Depois de make sim, leia sim.log.
  3. No sim.log, procure TEST_RESULT: PASS.
  4. No Verdi, observe d0, d1, d2, d3, sel e y.
Se você conseguir explicar o papel de mux4.sv, tb_mux4.sv, filelist.f e Makefile, este primeiro laboratório cumpriu o objetivo.
